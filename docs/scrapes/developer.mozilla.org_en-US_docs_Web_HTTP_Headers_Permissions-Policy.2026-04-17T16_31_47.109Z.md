# Permissions-Policy header

Limited availability

This feature is not Baseline because it does not work in some of the most widely-used browsers.

- [Learn more](https://developer.mozilla.org/en-US/docs/Glossary/Baseline/Compatibility)
- [See full compatibility](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#browser_compatibility)
- [Report feedback](https://survey.alchemer.com/s3/7634825/MDN-baseline-feedback?page=%2Fen-US%2Fdocs%2FWeb%2FHTTP%2FReference%2FHeaders%2FPermissions-Policy&level=not)

**Experimental:** **This is an [experimental technology](https://developer.mozilla.org/en-US/docs/MDN/Writing_guidelines/Experimental_deprecated_obsolete#experimental)**

Check the [Browser compatibility table](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#browser_compatibility) carefully before using this in production.

The HTTP **`Permissions-Policy`** [response header](https://developer.mozilla.org/en-US/docs/Glossary/Response_header) provides a mechanism to allow and deny the use of browser features in a document or within any [`<iframe>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/iframe) elements in the document.

For more information, see the main [Permissions Policy](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Permissions_Policy) article.

| Header type | [Response header](https://developer.mozilla.org/en-US/docs/Glossary/Response_header) |

## [Syntax](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy\#syntax)

httpCopy

```
Permissions-Policy: <directive>=<allowlist>
```

[`<directive>`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#directive)

The Permissions Policy directive to apply the `allowlist` to. See [Directives](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#directives) below for a list of the permitted directive names.

[`<allowlist>`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#allowlist)

An allowlist is a list of origins that takes one or more of the following values contained in parentheses, separated by spaces:

[`*` (wildcard)](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#sect)

The feature will be allowed in this document, and all nested browsing contexts (`<iframe>`s) regardless of their origin.

[`()` (empty allowlist)](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#sect_2)

The feature is disabled in top-level and nested browsing contexts. The equivalent for `<iframe>``allow` attributes is `'none'`.

[`self`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#self)

The feature will be allowed in this document, and in all nested browsing contexts (`<iframe>`s) in the same origin only. The feature is not allowed in cross-origin documents in nested browsing contexts. `self` can be considered shorthand for `https://your-site.example.com`. The equivalent for `<iframe>``allow` attributes is `self`.

[`src`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#src)

The feature will be allowed in this `<iframe>`, as long as the document loaded into it comes from the same origin as the URL in its [src](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/iframe#attributes) attribute. This value is only used in the `<iframe>``allow` attribute, and is the _default_`allowlist` value in `<iframe>`s.

[`"<origin>"`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#origin)

The feature is allowed for specific origins (for example, `"https://a.example.com"`). Origins should be separated by spaces. Note that origins in `<iframe>` allow attributes are not quoted.

The values `*` and `()` may only be used on their own, while `self` and `src` may be used in combination with one or more origins.

**Note:**
Directives have a default allowlist, which is always one of `*`, `self`, or `none` for the `Permissions-Policy` HTTP header, and governs the default behavior if they are not explicitly listed in a policy.
These are specified on the individual [directive reference pages](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy#directives). For `<iframe>``allow` attributes, the default behavior is always `src`.

Where supported, you can include wildcards in Permissions Policy origins. This means that instead of having to explicitly specify several different subdomains in an allowlist, you can specify them all in a single origin with a wildcard.

So instead of

httpCopy

```
("https://example.com" "https://a.example.com" "https://b.example.com" "https://c.example.com")
```

You can specify

httpCopy

```
("https://example.com" "https://*.example.com")
```

**Note:**`"https://*.example.com"` does not match `"https://example.com"`.

## [Directives](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy\#directives)

[`accelerometer`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/accelerometer)Experimental

Controls whether the current document is allowed to gather information about the acceleration of the device through the [`Accelerometer`](https://developer.mozilla.org/en-US/docs/Web/API/Accelerometer) interface.

[`ambient-light-sensor`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/ambient-light-sensor)Experimental

Controls whether the current document is allowed to gather information about the amount of light in the environment around the device through the [`AmbientLightSensor`](https://developer.mozilla.org/en-US/docs/Web/API/AmbientLightSensor) interface.

[`aria-notify`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/aria-notify)Experimental

Controls whether the current document is allowed to use the [`ariaNotify()`](https://developer.mozilla.org/en-US/docs/Web/API/Document/ariaNotify "ariaNotify()") method to fire [screen reader](https://developer.mozilla.org/en-US/docs/Glossary/Screen_reader) announcements.

[`attribution-reporting`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/attribution-reporting)Deprecated

Controls whether the current document is allowed to use the [Attribution Reporting API](https://developer.mozilla.org/en-US/docs/Web/API/Attribution_Reporting_API).

[`autoplay`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/autoplay)Experimental

Controls whether the current document is allowed to autoplay media requested through the [`HTMLMediaElement`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement) interface. When this policy is disabled and there were no user gestures, the [`Promise`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise) returned by [`HTMLMediaElement.play()`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement/play) will reject with a `NotAllowedError` [`DOMException`](https://developer.mozilla.org/en-US/docs/Web/API/DOMException). The autoplay attribute on [`<audio>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/audio) and [`<video>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/video) elements will be ignored.

[`bluetooth`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/bluetooth)Experimental

Controls whether the use of the [Web Bluetooth API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Bluetooth_API) is allowed. When this policy is disabled, the methods of the [`Bluetooth`](https://developer.mozilla.org/en-US/docs/Web/API/Bluetooth) object returned by [`Navigator.bluetooth`](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/bluetooth) will either return `false` or reject the returned [`Promise`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise) with a `SecurityError` [`DOMException`](https://developer.mozilla.org/en-US/docs/Web/API/DOMException).

[`browsing-topics`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/browsing-topics)DeprecatedNon-standard

Controls access to the [Topics API](https://developer.mozilla.org/en-US/docs/Web/API/Topics_API). Where a policy specifically disallows the use of the Topics API, any attempts to call the [`Document.browsingTopics()`](https://developer.mozilla.org/en-US/docs/Web/API/Document/browsingTopics) method or send a request with a [`Sec-Browsing-Topics`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Sec-Browsing-Topics) header will fail with a `NotAllowedError` [`DOMException`](https://developer.mozilla.org/en-US/docs/Web/API/DOMException).

[`camera`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/camera)Experimental

Controls whether the current document is allowed to use video input devices.
The [`Promise`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise) returned by [`getUserMedia()`](https://developer.mozilla.org/en-US/docs/Web/API/MediaDevices/getUserMedia "getUserMedia()") will reject with a `NotAllowedError` [`DOMException`](https://developer.mozilla.org/en-US/docs/Web/API/DOMException) if the permission is not allowed.

[`captured-surface-control`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/captured-surface-control)Experimental

Controls whether or not the document is permitted to use the [Captured Surface Control API](https://developer.mozilla.org/en-US/docs/Web/API/Screen_Capture_API/Captured_Surface_Control).
The promise returned by the API's main methods will reject with a `NotAllowedError` [`DOMException`](https://developer.mozilla.org/en-US/docs/Web/API/DOMException) if the permission is not allowed.

[`ch-ua-high-entropy-values`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/ch-ua-high-entropy-values)Experimental

Controls whether or not the document is permitted to use the [`NavigatorUAData.getHighEntropyValues()`](https://developer.mozilla.org/en-US/docs/Web/API/NavigatorUAData/getHighEntropyValues) method to retrieve high-entropy user-agent data.
If the permission is not allowed, the method will only return the `brands`, `mobile`, and `platform` low-entropy data.

[`compute-pressure`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/compute-pressure)Experimental

Controls access to the [Compute Pressure API](https://developer.mozilla.org/en-US/docs/Web/API/Compute_Pressure_API).

[`cross-origin-isolated`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/cross-origin-isolated)Experimental

Controls whether the current document can be treated as [cross-origin isolated](https://developer.mozilla.org/en-US/docs/Web/API/Window/crossOriginIsolated "cross-origin isolated").

[`deferred-fetch`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/deferred-fetch)Experimental

Controls the allocation of the top-level origin's [`fetchLater()` quota](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Deferred_Fetch#quotas).

[`deferred-fetch-minimal`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/deferred-fetch-minimal)Experimental

Controls the allocation of the shared cross-origin subframe [`fetchLater()` quota](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Deferred_Fetch#quotas).

[`display-capture`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/display-capture)Experimental

Controls whether or not the current document is permitted to use the [`getDisplayMedia()`](https://developer.mozilla.org/en-US/docs/Web/API/MediaDevices/getDisplayMedia "getDisplayMedia()") method to capture screen contents. When this policy is disabled, the promise returned by `getDisplayMedia()` will reject with a `NotAllowedError` [`DOMException`](https://developer.mozilla.org/en-US/docs/Web/API/DOMException) if permission is not obtained to capture the display's contents.

[`encrypted-media`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/encrypted-media)Experimental

Controls whether the current document is allowed to use the [Encrypted Media Extensions API](https://developer.mozilla.org/en-US/docs/Web/API/Encrypted_Media_Extensions_API) (EME). When this policy is disabled, the [`Promise`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise) returned by [`Navigator.requestMediaKeySystemAccess()`](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/requestMediaKeySystemAccess) will reject with a `SecurityError` [`DOMException`](https://developer.mozilla.org/en-US/docs/Web/API/DOMException).

[`fullscreen`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/fullscreen)Experimental

Controls whether the current document is allowed to use [`Element.requestFullscreen()`](https://developer.mozilla.org/en-US/docs/Web/API/Element/requestFullscreen). When this policy is disabled, the returned [`Promise`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise) rejects with a [`TypeError`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/TypeError).

[`gamepad`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/gamepad)Experimental

Controls whether the current document is allowed to use the [Gamepad API](https://developer.mozilla.org/en-US/docs/Web/API/Gamepad_API).
When this policy is disabled, calls to [`Navigator.getGamepads()`](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/getGamepads) will throw a `SecurityError` [`DOMException`](https://developer.mozilla.org/en-US/docs/Web/API/DOMException), and the [`gamepadconnected`](https://developer.mozilla.org/en-US/docs/Web/API/Window/gamepadconnected_event "gamepadconnected") and [`gamepaddisconnected`](https://developer.mozilla.org/en-US/docs/Web/API/Window/gamepaddisconnected_event "gamepaddisconnected") events will not fire.

[`geolocation`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/geolocation)Experimental

Controls whether the current document is allowed to use the [`Geolocation`](https://developer.mozilla.org/en-US/docs/Web/API/Geolocation) Interface. When this policy is disabled, calls to [`getCurrentPosition()`](https://developer.mozilla.org/en-US/docs/Web/API/Geolocation/getCurrentPosition "getCurrentPosition()") and [`watchPosition()`](https://developer.mozilla.org/en-US/docs/Web/API/Geolocation/watchPosition "watchPosition()") will cause those functions' callbacks to be invoked with a [`GeolocationPositionError`](https://developer.mozilla.org/en-US/docs/Web/API/GeolocationPositionError) code of `PERMISSION_DENIED`.

[`gyroscope`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy/gyroscope)Experimental

## [Examples](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy\#examples)

### [Basic usage](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy\#basic_usage)

#### Permissions-Policy header

To allow all origins access to geolocation, you would do this:

httpCopy

```
Permissions-Policy: geolocation=*
```

Or to allow access to a subset of origins, you'd do this:

httpCopy

```
Permissions-Policy: geolocation=(self "https://a.example.com" "https://b.example.com")
```

Several features can be controlled at the same time by sending the header with a comma-separated list of policies, or by sending a separate header for each policy.

For example, the following are equivalent:

httpCopy

```
Permissions-Policy: picture-in-picture=(), geolocation=(self https://example.com/), camera=*

Permissions-Policy: picture-in-picture=()
Permissions-Policy: geolocation=(self https://example.com/)
Permissions-Policy: camera=*
```

#### iframes

For an `<iframe>` to have a feature enabled its allowed origin must also be in the allowlist for the parent page. Because of this [inheritance behavior](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Permissions_Policy#inheritance_of_policies_for_embedded_content), it is a good idea to specify the widest acceptable support for a feature in the HTTP header, and then specify the subset of support you need in each `<iframe>`.

To allow all origins access to geolocation, you would do this:

htmlCopy

```
<iframe src="https://example.com" allow="geolocation *"></iframe>
```

To apply a policy to the current origin and others, you'd do this:

htmlCopy

```
<iframe
  src="https://example.com"
  allow="geolocation 'self' https://a.example.com https://b.example.com"></iframe>
```

This is important: By default, if an `<iframe>` navigates to another origin, the policy is not applied to the origin that the `<iframe>` navigates to. By listing the origin that the `<iframe>` navigates to in the `allow` attribute, the Permissions Policy that was applied to the original `<iframe>` will be applied to the origin the `<iframe>` navigates to.

Several features can be controlled at the same time by including a semi-colon-separated list of policy directives inside the `allow` attribute.

htmlCopy

```
<iframe
  src="https://example.com"
  allow="geolocation 'self' https://a.example.com https://b.example.com; fullscreen 'none'"></iframe>
```

It is worth giving the `src` value a special mention. We mentioned above that using this allowlist value will mean that the associated feature will be allowed in this `<iframe>`, as long as the document loaded into it comes from the same origin as the URL in its [src](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/iframe#attributes) attribute. This value is the _default_`allowlist` value for features listed in `allow`, so the following are equivalent:

htmlCopy

```
<iframe src="https://example.com" allow="geolocation 'src'"></iframe>
<iframe src="https://example.com" allow="geolocation"></iframe>
```

### [Denying access to powerful features](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy\#denying_access_to_powerful_features)

SecureCorp Inc. wants to disable Microphone (for example [`MediaDevices.getUserMedia()`](https://developer.mozilla.org/en-US/docs/Web/API/MediaDevices/getUserMedia)) and [`Geolocation`](https://developer.mozilla.org/en-US/docs/Web/API/Geolocation) APIs in its application. It can do so using the following response header:

httpCopy

```
Permissions-Policy: microphone=(), geolocation=()
```

By specifying `()` for the origin list, the specified features will be disabled for all browsing contexts (this includes all `<iframe>`s), regardless of their origin.

### [Combining HTTP header and `<iframe>` policies](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Permissions-Policy\#combining_http_header_and_iframe_policies)

For example, let's say that we wanted to enable geolocation usage on our own origin, and in embedded content coming from our trusted ad network. We could set up the page-wide Permissions Policy like this:

httpCopy

```
Permissions-Policy: geolocation=(self https://trusted-ad-network.com)
```

Over in our ad `<iframe>`s, we could set access to the `https://trusted-ad-network.com` origin like this:

htmlCopy

```
<iframe src="https://trusted-ad-network.com" allow="geolocation"></iframe>
```

If a different origin ended up getting loaded into `<iframe>`, it would not have access to geolocation:

htmlCopy

```
<iframe src="https://rogue-origin-example.com" allow="geolocation"></iframe>
```
